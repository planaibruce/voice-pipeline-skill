# Voice Pipeline Skill - Core Vision & Builder Prompt

**NOTE: This file contains the foundational vision and instructions for the AI assistant (like Antigravity/Opencode) that BUILDS and MAINTAINS this skill. It is NOT the runtime prompt for the OpenClaw agent.**

## Big Vision
The "Voice Pipeline" skill is designed to be the ultimate, fully local audio processing and transcription pipeline for Arch Linux (and macOS experimentally). It ensures absolute privacy by never falling back to cloud APIs.

## Core Principles
1. **Local-Only:** Everything runs locally. Whisper for transcription, ClearVoice (MossFormer2) for speech enhancement. Zero cloud fallback.
2. **Seamless Audio Prep:** The pipeline aggressively removes silence and background noise to create a highly compressed, clean MP3 for the user, while preparing a carefully optimized WAV for Whisper to ensure maximum transcription accuracy (without hallucinating).
3. **Resilience & Discovery:** The script must auto-detect runtimes (like `~/.venvs/...` or `~/.openclaw/...`). It should never auto-install dependencies. Instead, if something is missing, it provides copy-pasteable OS-specific install commands.
4. **Agent-Ready Output:** The bash script handles the heavy lifting and outputs clear paths (`RESULT|...`). The runtime OpenClaw agent is responsible for providing a semantic summary and attaching the files using the `MEDIA:<path>` syntax.

## Development Guidelines
- Maintain pure bash (`set -euo pipefail`) without external dependencies beyond standard Unix tools, Python, and FFmpeg.
- Ensure robust fallback logic (e.g., CUDA Out-Of-Memory gracefully falls back to smaller models or CPU if allowed).
- Sort outputs cleanly by timestamp in `~/voice-transcripts`.
